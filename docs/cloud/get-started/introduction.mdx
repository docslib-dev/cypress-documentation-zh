## 优势

[Cypress Cloud](https://on.cypress.io/about-cypress-cloud?utm_medium=cloud-benefits&utm_source=docs.cypress.io&utm_term=&utm_content=Cypress+Cloud) 是我们为企业准备的开源 Cypress 应用伴侣，让您能够：

- [查看和调试历史测试结果](#View-and-debug-past-test-results) 来自您的 CI 环境。
- [分析和诊断](#Analyze-and-diagnose-test-health) 测试健康状况。
- [减少测试时间并节省成本](#Reduce-test-time--CI-costs) 通过智能编排多台机器的未来运行。
- [与您喜爱的开发者工具集成](#Integrate-with-your-favorite-tools) 以帮助简化工作流程。
- [直接在 Cypress 应用中查看 CI 运行和测试健康状况](#Cypress-app-integration)。
- [按需配置 Cypress Cloud](#Flexible-enterprise-configuration-and-single-sign-on) 并启用企业单点登录。
- 添加高级解决方案如 [UI Coverage](https://on.cypress.io/ui-coverage-trial?utm_medium=cloud-benefits&utm_source=docs.cypress.io&utm_term=&utm_content=UI+Coverage) 和 [Cypress Accessibility](http://on.cypress.io/accessibility-trial?utm_medium=cloud-benefits&utm_source=docs.cypress.io&utm_term=&utm_content=Cypress+Accessibility)，无需代码、设置或影响测试运行即可获得测试和应用质量的新洞察。

### 查看和调试历史测试结果

每次测试运行都存储在 Cypress Cloud 中，您可以在[最新运行](/cloud/features/recorded-runs)页面查看历史结果和应用的当前状态。

使用[<Logo src="/img/cloud/features/test-replay/replay-icon.svg"  alt="Replay Icon"/> 测试回放](/cloud/features/test-replay)功能，以完整调试能力回放测试执行过程。

或查看每个测试的[命令日志、截图、视频回放、堆栈跟踪和 CI 日志](/cloud/features/recorded-runs#Test-detail-sidebar)。快速识别 CI 中的测试失败只需点击一下。

### 分析和诊断测试健康状况

查看每次[运行的概览](/cloud/features/recorded-runs#Overview-tab)，比较历史运行并分析趋势。您可以快速分析设置中可能引入问题趋势的变化，并通过[不稳定测试管理](/cloud/features/flaky-test-management)识别不可靠的测试。

### 减少测试时间和 CI 成本

[智能编排](/cloud/features/smart-orchestration/overview)让您可以在 CI 环境中同时在多台机器上运行测试，而 Cypress Cloud 协调运行器并[平衡测试负载](/cloud/features/smart-orchestration/load-balancing#Balance-strategy) - 无需设置！您可以使用[规范优先级](/cloud/features/smart-orchestration/spec-prioritization)优先运行最近失败的规范，以更早发现问题，并通过[自动取消](/cloud/features/smart-orchestration/run-cancellation)在失败时取消整个测试运行以节省资源使用。如果需要，您还可以从 Cypress Cloud [手动取消进行中的运行](/cloud/features/recorded-runs#Run-cancellation)。

<DocsImage
  src="/img/cloud/get-started/introduction/orchestration-diagram.png"
  alt="串行和并行测试配置对比图"
/>

Cypress Cloud 的[规范优先级](/cloud/features/smart-orchestration/spec-prioritization)让您优先运行最近失败的规范，使它们在下次运行时更快。[自动取消](/cloud/features/smart-orchestration/run-cancellation)功能会在失败时取消整个测试运行，节省时间和资源使用。如果需要，您还可以从 Cypress Cloud [手动取消进行中的运行](/cloud/features/recorded-runs#Run-cancellation)。

### 与您喜爱的工具集成

#### 源代码控制提供商

Cypress Cloud 与 [GitHub](/cloud/integrations/source-control/github)、[GitLab](/cloud/integrations/source-control/gitlab) 和 [Bitbucket](/cloud/integrations/source-control/bitbucket) 集成，通过阻止失败代码进入主分支提供坚如磐石的可靠性。使用状态检查阻止提交合并，直到您的 Cypress 测试通过，并通过拉取请求评论直接在 PR 中展示测试结果，包括测试运行统计、具体失败详情和 Cypress Cloud 结果的深度链接以便快速调试。

[商业和企业计划](https://www.cypress.io/pricing/)还包括与自托管的 [GitHub](https://docs.github.com/en/enterprise-server@3.5/admin/overview/about-github-enterprise-server) 和 [GitLab](https://docs.gitlab.com/ee/subscriptions/self_managed/) 实例的安全集成。

  src="/img/cloud/integrations/github/status-checks-per-spec.jpg"

#### 团队沟通

#### 问题管理

通过 [Jira](/cloud/integrations/jira) 集成让整个团队保持最新状态。您可以直接从特定测试失败[创建双向链接的 Jira 工单](/cloud/integrations/jira#Creating-a-Jira-issue-for-a-test-case)。

### Cypress 应用集成

[Cypress 应用](/cloud/features/recorded-runs#Cypress-App)直接与 Cypress Cloud 集成，为开发者提供最新的测试结果而无需切换工具。这提供了关于 CI 中新失败测试的通知，然后您可以在本地环境中立即重新运行它们进行调试。

### 灵活的企业配置和单点登录

您可以使用我们灵活的管理功能按需配置 Cypress Cloud，将[项目](/cloud/account-management/projects)分组到多个[组织](/cloud/account-management/organizations)（如果有大量项目），检查[使用情况](/cloud/account-management/billing-and-usage)，并管理[用户和权限](/cloud/account-management/users#User-roles)。

我们还为[商业或企业计划](https://www.cypress.io/pricing/)的团队提供 [SSO 集成](/cloud/account-management/enterprise-sso)。
