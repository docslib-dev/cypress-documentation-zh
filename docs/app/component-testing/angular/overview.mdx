# Angular组件测试

:::info

##### <Icon name="question-circle" color="#4BBFD2" /> 你将学到

- 如何在Angular中设置组件测试

## 框架支持

Cypress组件测试支持Angular 17.2.0及以上版本。

## 教程

访问[入门指南](/app/component-testing/get-started)，获取如何为任何项目添加组件测试并编写第一个测试的逐步教程。

## 安装

要在Angular中开始使用Cypress组件测试，请将Cypress安装到你的项目中：

<CypressInstallCommands />

打开Cypress：

<CypressOpenCommands />

<DocsImage
  src="/img/app/component-testing/select-test-type.jpg"
  caption="选择组件测试"
/>

Cypress应用将引导你完成项目配置。

:::info

如需创建组件测试的逐步指南，请参考[入门](/app/component-testing/get-started)指南。

关于使用方法和示例，请访问[Angular示例](/app/component-testing/angular/examples)指南。

:::

## 框架配置

Cypress组件测试与`@angular/cli`项目开箱即用。在设置过程中，Cypress会自动检测到你的项目是Angular并正确配置。以下示例仅供参考。

### Angular CLI配置

:::cypress-config-example

```ts
{
  component: {
    devServer: {
      framework: 'angular',
      bundler: 'webpack',
    },
    specPattern: '**/*.cy.ts',
  },
}
```

:::

#### 选项API

你也可以使用`options` API为`devServer`提供项目特定的配置。`devServer`配置接收一个`options`属性：

```ts title=cypress.config.ts
import { defineConfig } from 'cypress'

export default {
  component: {
    devServer: {
      framework: 'angular',
      bundler: 'webpack',
      options: {
        projectConfig: {
          root: '',
          sourceRoot: 'apps/my-app',
          buildOptions: {
            outputPath: 'dist/my-app',
            index: 'apps/my-app/src/index.html',
            main: 'apps/my-app/src/main.ts',
            polyfills: 'apps/my-app/src/polyfills.ts',
            tsConfig: 'apps/my-app/tsconfig.app.json',
            inlineStyleLanguage: 'scss',
            assets: ['apps/my-app/src/favicon.ico', 'apps/my-app/src/assets'],
            styles: ['apps/my-app/src/styles.scss'],
            scripts: [],
            buildOptimizer: false,
            optimization: false,
            vendorChunk: true,
            extractLicenses: false,
            sourceMap: true,
            namedChunks: true,
          },
        },
      },
    },
  },
}
```
