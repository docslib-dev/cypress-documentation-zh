# 快速入门

UI Coverage 无需设置或代码插桩即可立即使用。如果您使用 Test Replay 将测试运行记录到 Cypress Cloud，就可以开始使用 UI Coverage。
您将通过销售团队的个性化支持和现有测试数据开始免费试用，立即查看所有 Cypress 项目中存在的测试覆盖缺口。
之后，您可以通过灵活的配置选项轻松自定义报告以满足需求。

## 1. 申请试用

开始免费试用 UI Coverage。您将获得个性化演示，使用现有测试数据查看所有 Cypress 项目中存在的测试覆盖缺口。

## 2. 记录测试运行

UI Coverage 使用 [Test Replay](/cloud/features/test-replay) 数据生成报告，需要 Cypress v13+ 版本。开始使用 UI Coverage 无需额外配置或代码插桩。

使用 Test Replay [记录测试运行](/cloud/get-started/setup) 到 Cypress Cloud 即可开始使用 UI Coverage。

## 3. 查看 UI Coverage 结果

记录测试运行后，您可以在 Cypress Cloud 的 **UI Coverage** 选项卡中查看 UI Coverage 结果。

## 4. 自定义结果

UI Coverage 提供灵活的配置选项来自定义报告。您可以配置：

- [属性过滤器](/ui-coverage/configuration/attributefilters)：指定不应用于识别和分组元素的属性及其值的模式
- [元素过滤器](/ui-coverage/configuration/elementfilters)：指定应从 UI Coverage 中排除的元素选择器
- [元素组](/ui-coverage/configuration/elementgroups)：将相关元素分组
- [元素](/ui-coverage/configuration/elements)：指定用于唯一标识元素的选择器
- [重要属性](/ui-coverage/configuration/significantattributes)：优先识别和分组特定属性
- [视图过滤器](/ui-coverage/configuration/viewfilters)：指定应从 UI Coverage 中排除的 URL 模式。排除 URL 也会排除指向该 URL 的所有链接。
- [视图](/ui-coverage/configuration/views)：指定表示视图的 URL 模式

要添加或修改项目配置，请导航至项目设置中的 **App Quality** 选项卡。
