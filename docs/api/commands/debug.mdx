# debug

设置`debugger`并记录前一个命令的返回结果。

:::caution

使用`.debug()`触发断点时，需要保持开发者工具处于打开状态。

:::

## 语法

```javascript
.debug()
.debug(options)

// ---或---

cy.debug()
cy.debug(options)
```

### 用法

<Icon name="check-circle" color="green" /> **正确用法**

```javascript
cy.debug().getCookie('app') // 在命令开始时暂停调试
cy.get('nav').debug() // 调试`get`命令的返回结果
```

### 参数

<Icon name="angle-right" /> **options _(对象)_**

传入选项对象以改变`.debug()`的默认行为。

| 选项    | 默认值  | 描述                                                                         |
| ------- | ------- | --------------------------------------------------------------------------- |
| `log`   | `true`  | 在[命令日志](/app/core-concepts/open-mode#Command-Log)中显示该命令          |

<HeaderYields />

- `.debug()`返回与前一个命令相同的主题对象。
- `.debug()`是一个查询命令，可以安全地链式调用其他命令。

## 示例

### 调试

#### 在`.get()`后使用debugger暂停

```javascript
cy.get('a').debug().should('have.attr', 'href')
```

## 规则

<HeaderRequirements />

- `.debug()`可以链式调用在`cy`或其他命令之后。

<HeaderAssertions />

- `.debug()`是一个实用命令。
- `.debug()`不会运行断言。断言会直接通过，就像该命令不存在一样。

<HeaderTimeouts />

- `.debug()`不会超时。

## 命令日志
